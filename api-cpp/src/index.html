R-GMA C++ API

This is the user API for R-GMA, a relational implementation of the Grid Monitoring Architecture from the OGF. It creates the impression of inserting data into and fetching it from a distributed RDBMS.

Assuming you already have access to a VDB then to create a table definition or modify it  create a Schema object for that VDB.

To publish data create a PrimaryProducer and insert data into it.

To consume data create a Consumer with your query and pop TupleSets from it. 

Many calls are able to throw exceptions. These are subclasses of the RGMAException and should be handled as explained below:

RGMATemporaryException
This can be thrown for a number of reasons including network problems. A sensible action is to to try the call again after a short interval.
PermanentException
This is for errors such as configuration or user errors. The source of the problem should be investigated.

The R-GMA User Guide for C++ Programmers is the best starting point for using this API. Also please see the autogenerated documentation.
